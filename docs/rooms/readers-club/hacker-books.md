# Hacker Books

Technical security books that build real capability. Not certification guides. Not compliance checklists. The books that practitioners reference when the documentation isn't enough.

**Exploit Development & Reverse Engineering**  
*The Art of Exploitation* (Erickson) — Stack/heap, shellcode, format strings, ASLR bypass. Hands-on with GDB.  
*Practical Malware Analysis* (Sikorski & Honig) — Static/dynamic analysis, IDA Pro, OllyDbg, anti-debugging, packing.  
*Reverse Engineering for Beginners* (Yurichev) — Free, comprehensive, from assembly to decompilation patterns.  

**Cryptography**  
*Serious Cryptography* (Aumasson) — Modern crypto engineering. AEAD, key derivation, TLS, post-quantum. No math masturbation.  
*Cryptography Engineering* (Ferguson, Schneier, Kohno) — Protocol design, side channels, random number generation.  
*Real-World Cryptography* (Wong) — TLS, Noise, WireGuard, Signal protocol, password-authenticated key exchange.  

**Network Security & Protocols**  
*TCP/IP Illustrated, Vol 1* (Stevens) — The protocol bible. Packet captures, not theory.  
*The Tangled Web* (Zalewski) — Browser security model, same-origin policy, CSRF, XSS, CSP.  
*SSL and TLS* (Rescorla) — Protocol internals, handshakes, resumption, extensions.  

**System Internals**  
*Operating Systems: Three Easy Pieces* (Arpaci-Dusseau) — Virtualization, concurrency, persistence. Free. Excellent.  
*Linux Kernel Development* (Love) — Scheduler, memory, VFS, block layer, interrupts.  
*Windows Internals* (Russinovich) — The Windows equivalent. Object manager, ALPC, ETW, kernel pooling.
